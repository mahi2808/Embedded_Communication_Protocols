SPI (Serial Peripheral Interface)
1. Introduction
SPI stands for: Serial Peripheral Interface
It is a: Synchronous Serial Communication Protocol
developed by: Motorola
SPI is used for communication between: Microcontrollers, EEPROM, Flash Memory, ADC, DAC, Displays (LCD/TFT/OLED), Sensors

2. Why SPI is Required?
Suppose MCU wants to communicate with: ADC, DAC, EEPROM, Display
Communication must be: Fast, Reliable, Simple
SPI provides: High Speed Communication compared to I²C.

3. SPI Communication Type Serial Communication
Data is transferred: One Bit at a Time
Example: 10110010
Synchronous Communication Communication uses a clock signal.
Clock synchronizes: Data Transmission & Data Reception

4. Master and Slave
Master
Responsible for:
Generating Clock
Selecting Slave
Starting Communication
Sending and Receiving Data
Examples: STM32, MSPM0, PIC, AVR, ESP32

Slave Responds when selected by Master.
Examples: ADC, DAC, EEPROM, Display, Sensor.

5. SPI Signals:- SPI uses four signals.
SCLK (Serial Clock):- Generated by: Master Used to synchronize communication.
MOSI:- Meaning: Master Out Slave In, Data flow: Master → Slave
MISO:- Meaning: Master In Slave Out, Data flow: Slave → Master
SS / CS:- Meaning: Slave Select Chip Select, Used to select a specific slave Usually: Active LOW

6. SPI Connection
           Master

MOSI -----------------> MOSI

MISO <----------------- MISO

SCLK -----------------> SCLK

CS   -----------------> CS

7. Data Flow Direction
Transmit:- Master → MOSI → Slave
Receive:- Slave → MISO → Master
Both happen simultaneously.

8. Full Duplex Communication
SPI supports: Full Duplex
Meaning: Transmit and Receive at the same time

Example:
Master sends : 0xA5
Slave sends  : 0x3C
Both transfers occur during the same clock pulses.

9. Basic Communication Sequence
Step 1 
Master selects slave:
CS = LOW

Step 2
Master generates clock:
SCLK

Step 3
Data transfer occurs:
MOSI → Master to Slave
MISO → Slave to Master

Step 4
Communication ends:
CS = HIGH
Slave becomes inactive.

11. SPI Clock Operation
What is SCLK?
SCLK stands for: Serial Clock
It is generated by: Master
The clock synchronizes data transfer between master and slave.

Why is Clock Required?
Suppose master wants to send: 10110010
Question: When should the slave read each bit?
Answer: Using SCLK
The clock tells both devices exactly when a bit is valid.

Basic Clock Example
SCLK : _|‾|_|‾|_|‾|_|‾|_
Each pulse transfers: 1 bit

Example Data Transfer
Master wants to send:
MOSI:   1   0   1   1   0   0   1   0
SCLK: _|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_|‾|_
Therefore: 8 Clock Pulses are required to transfer: 8 Bits

# Data Sampling
During communication: Master places data on MOSI and Slave reads data at a specific clock edge.
Similarly: Slave places data on MISO and Master reads data at the same clock edge.

# Clock Edges
Every clock pulse has:
Rising Edge: LOW → HIGH
Example:

_|‾
 ↑
Rising Edge

Falling Edge: HIGH → LOW
Example:

‾|_
 ↑
Falling Edge

# Which Edge is Used?
Depends on SPI Mode.
Some devices: Sample on Rising Edge
Others: Sample on Falling Edge
This is controlled by: CPOL & CPHA

Simple Example
Assume data is sampled on rising edge.
SCLK : _|‾|_|‾|_|‾|_
MOSI :  1   0   1
At each rising edge: Slave reads data

# Full Duplex Operation
While master sends data on: MOSI
slave can simultaneously send data on: MISO

Example:
Master → 10110010
Slave  → 01010111
Both transfers happen using the same clock pulses.

# Clock Ownership
In SPI: Master always generates clock
Slave never generates SCLK.
If clock stops: Data transfer stops because communication is synchronized by the clock.

12. Shift Register Concept
What is a Shift Register?
A shift register is a hardware register that can:
Store Data, Shift Data Bit-by-Bit
SPI communication is based on shift registers.
Both: Master & Slave contain SPI shift registers.

Example
Suppose:
Master wants to send: 0xA5
Binary: 10100101

Slave wants to send: 0x3C
Binary: 00111100

# Before communication:
Master Shift Register
10100101

Slave Shift Register
00111100

# Communication Begins
Master: CS = LOW
Clock starts: SCLK

Clock Pulse #1
Master sends: 1 on MOSI.
Slave sends: 0 on MISO.

Both devices receive one bit.
Registers shift by one position.

Clock Pulse #2
Master sends next bit: 0
Slave sends next bit: 0
Again both registers shift.
Process Continues For every clock pulse: 1 bit transmitted & 1 bit received simultaneously.

After 8 Clock Pulses
Master has received: 00111100
Slave has received: 10100101
Result: Data exchanged completely

# Visualization
Before Transfer:
Master Register 10100101
Slave Register  00111100

After Transfer:
Master Register 00111100
Slave Register  10100101
They effectively exchange data.

# Why is Shift Register Needed?
Without a shift register: Bit-by-bit transfer would be difficult
Shift registers allow hardware to: Transmit, Receive & Shift automatically on every clock pulse.

# Read Operation Example
Suppose master wants to read a sensor.
Master must generate clocks.
Therefore it sends: Dummy Data
Example: 0x00 or 0xFF on MOSI.
Meanwhile sensor sends actual data on MISO.
Example:
Master MOSI → 0x00
Sensor MISO → 0x56

After transfer: Master receives 0x56
This is why SPI reads usually involve sending dummy bytes.

# Write Operation Example
Suppose master writes to EEPROM.
Master sends: Command, Address & Data on MOSI.
EEPROM may simultaneously send: Status Bits or Dummy Data on MISO.
Master may ignore them.

Key Concept
During SPI communication: Transmit and Receive always occur together, Even if one direction contains only dummy data.

14. SPI Modes (CPOL & CPHA)
This is one of the most important SPI topics.
Until now we learned:
Master generates SCLK
Data transferred on MOSI/MISO

But we never answered:
When should data be sampled?
When should data change?

SPI Modes solve this problem.
Why SPI Modes are Required?
Different SPI devices use different timing requirements.
Some devices: Read data on Rising Edge
Others: Read data on Falling Edge
Some devices keep clock idle LOW.
Others keep clock idle HIGH.

Therefore SPI defines:
CPOL
CPHA
Two Parameters

1. CPOL
CPOL means: Clock Polarity
It defines: Idle state of SCLK

Meaning:
What is the clock level when communication is not happening?
CPOL = 0
Idle clock: LOW

Example:
_____|‾|_|‾|_|‾|_
Clock rests at LOW.

CPOL = 1
Idle clock: HIGH

Example:
‾‾‾‾|_|‾|_|‾|_
Clock rests at HIGH.

2. CPHA
CPHA means: Clock Phase

It defines:
Which clock edge is used to sample data
CPHA = 0
Sample on: First Clock Edge

CPHA = 1
Sample on: Second Clock Edge

What is First Edge?
Depends on CPOL.

If CPOL = 0
Clock starts LOW.
First transition: LOW → HIGH

Therefore:
First Edge = Rising Edge
Second Edge = Falling Edge

If CPOL = 1
Clock starts HIGH.
First transition: HIGH → LOW

Therefore:
First Edge = Falling Edge
Second Edge = Rising Edge

# Combining CPOL and CPHA
SPI supports: 4 Modes
because:
2 CPOL options
×
2 CPHA options
=
4 Modes

SPI Mode Table
| Mode   | CPOL | CPHA |
| ------ | ---- | ---- |
| Mode 0 | 0    | 0    |
| Mode 1 | 0    | 1    |
| Mode 2 | 1    | 0    |
| Mode 3 | 1    | 1    |

Important Rule
MCU and Slave must use: Same SPI Mode

Otherwise:
Wrong Data
Missing Bits
Communication Failure
may occur.

Example
Suppose sensor datasheet says: SPI Mode 0
Then MCU must configure:
CPOL = 0
CPHA = 0

Now only ADC responds.

Why Doesn't MISO Conflict? Good question.
Suppose:
EEPROM Active
ADC Inactive
Display Inactive

Inactive slaves place MISO in: High Impedance (Hi-Z) state.

Meaning: Disconnected Electrically from the bus.
Therefore only the selected slave drives MISO.

High Impedance State
Think of it as: Output Disabled

Example:
Selected Slave MISO = 1 or 0
Unselected Slave MISO = Hi-Z
No bus contention occurs.

Result:
Bus Contention
Possible effects:

Communication Errors
Excess Current

Rule
Normally:
Only One CS should be LOW at a time.

25. Dummy Byte Concept
This is one of the most important SPI concepts.
You already know: SPI cannot receive data without clock pulses.

And: Only Master generates clock.
This is exactly why Dummy Bytes are used.

What is a Dummy Byte?
A dummy byte is: A transmitted byte whose value is not important.
Purpose: Generate clock pulses for SPI communication.

Why is Dummy Byte Needed?
Suppose EEPROM wants to send: 0x56 to the master.
Question: How will EEPROM send data?
Answer: Using SCLK pulses.
But: EEPROM cannot generate SCLK.
Only master can.
Therefore master must generate clocks.

How Does Master Generate Clocks?
SPI hardware generates clock while transmitting.
Example:
Master transmits 8 bits
↓
8 clock pulses generated

Solution
Transmit: Dummy Byte
Example:
0x00 or 0xFF

Example
Master wants to read EEPROM.
Frame:
CS LOW
READ Command
Address
0x00   ← Dummy Byte
Receive Data
CS HIGH

During Dummy Transfer
MOSI Master sends: 0x00
MISO EEPROM sends:0x56
Result Master receives: 0x56

Is Dummy Data Stored Anywhere?
No. Slave usually ignores it.
Example: MOSI = 0x00
EEPROM: Ignore MOSI because it is already in: READ mode

Why 0x00? Simply because:
Easy value and commonly used.
Why 0xFF? Also common.
Some devices prefer: 0xFF
during reads. Datasheet may specify.

# Important Point
Dummy byte value usually: Does not matter
because its purpose is only: Clock Generation
