1. What is SPI?
SPI (Serial Peripheral Interface) is a synchronous serial communication protocol used for high-speed communication between a master and one or more slave devices.

2. How many lines are used in SPI?
4 Lines SCLK, MOSI, MISO, CS/SS

3. Who generates the SPI clock?
Master generates the SCLK signal.

4. What is MOSI?
Master Out Slave In, Data travels from Master to Slave.

5. What is MISO?
Master In Slave Out, Data travels from Slave to Master.

6. What is CS/SS?
CS (Chip Select) or SS (Slave Select) is used by the master to select a specific slave device for communication.

7. Is SPI Full Duplex or Half Duplex?
Full Duplex, because transmission and reception occur simultaneously.

8. Why is SPI faster than I²C?
SPI is faster because it does not require address transmission, ACK/NACK bits, and supports full-duplex communication at higher clock frequencies.

9. Which protocol has less overhead?
SPI, because it does not use address or ACK/NACK fields.

10. What is the main disadvantage of SPI?
More communication lines are required.

11. What is SCLK in SPI?
SCLK (Serial Clock) is the clock signal generated by the master to synchronize SPI communication.

12. Who generates the SPI clock?
Master

13. How many clock pulses are required to transfer one byte?
8 clock pulses, because one pulse transfers one bit.

14. What are the two edges of a clock signal?
Rising Edge  = LOW → HIGH
Falling Edge = HIGH → LOW

15. Can the slave generate SCLK?
No, Only the master generates the SPI clock.

16. What is a shift register?
A shift register is a register that stores data and shifts it one bit at a time on each clock pulse.

17. Why does SPI use shift registers?
To enable automatic bit-by-bit transmission and reception synchronized with the clock.

18. What happens during one SPI clock pulse?
One bit transmitted & One bit received, simultaneously.

19. Why are dummy bytes sent during SPI read operations?
The master must generate clock pulses for the slave to send data. Therefore dummy bytes (such as 0x00 or 0xFF) are transmitted while reading.

20. Can SPI receive data without transmitting anything?
No. The master must generate clock pulses, which normally means transmitting a byte (often a dummy byte).

21. What is transferred during one SPI clock pulse?
1 bit on MOSI & 1 bit on MISO, simultaneously.

22. Why is SPI called Full Duplex?
Because transmission and reception occur simultaneously during the same clock pulses.

23. How many clock pulses are required to exchange one byte?
8 clock pulses

24. Can the master receive data while transmitting?
This is the normal SPI operation

25. What is CPOL?
Clock Polarity. It defines the idle state of the SPI clock.

26. What is CPHA?
Clock Phase. It defines which clock edge is used to sample data.

27. How many SPI modes are available?
4 Modes
Mode 0, Mode 1, Mode 2 and Mode 3.

28. Why must master and slave use the same SPI mode?
Because both devices must sample and change data on the same clock edges. Otherwise communication errors occur

29. What are the CPOL and CPHA values for SPI Mode 0?
CPOL = 0
CPHA = 0

30. What is the idle clock level in Mode 0?
LOW

31. On which edge is data sampled in Mode 0?
Rising Edge

32. On which edge is data changed in Mode 0?
Falling Edge

33. What are the CPOL and CPHA values for Mode 1?
CPOL = 0
CPHA = 1

34. What is the idle clock level in Mode 1?
LOW

35. On which edge is data sampled in Mode 1?
Falling Edge

36. On which edge is data changed in Mode 1?
Rising Edge

37. What are the CPOL and CPHA values for Mode 2?
CPOL = 1
CPHA = 0

38. What is the idle clock level in Mode 2?
HIGH

39. What is the first edge in Mode 2?
Falling Edge, because CPOL = 1.

40. On which edge is data sampled in Mode 2?
Falling Edge

41. On which edge is data changed in Mode 2?
Rising Edge

42. What are the CPOL and CPHA values for Mode 3?
CPOL = 1
CPHA = 1

43. What is the idle clock level in Mode 3?
HIGH

44. On which edge is data sampled in Mode 3?
Rising Edge

45. On which edge is data changed in Mode 3?
Falling Edge

46. What is MSB?
Most Significant Bit
Highest-value bit of a byte.

47. What is LSB?
Least Significant Bit
Lowest-value bit of a byte.

48. What is MSB First transmission?
Data transmission starts from Bit7 and ends at Bit0.

49. What is LSB First transmission?
Data transmission starts from Bit0 and ends at Bit7.

50. Does SPI support both MSB First and LSB First?
Yes. SPI can be configured for either bit order.

51. What is an SPI frame?
An SPI frame is the complete sequence of bytes transferred between CS LOW and CS HIGH.

52. What marks the beginning of an SPI transaction?
CS = LOW


53. What marks the end of an SPI transaction?
CS = HIGH


54. Does SPI have START and STOP conditions?
No, SPI uses the CS signal to indicate transaction boundaries.


55. Does SPI define command and address formats?
No, The command and address format are defined by the slave device datasheet.

56. What is the purpose of CS in SPI?
CS is used by the master to select a specific slave device for communication.

57. Is CS usually Active HIGH or Active LOW?
Active LOW

58. What happens when CS becomes LOW?
The slave device becomes active and participates in communication.


59. What happens when CS becomes HIGH?
The slave device becomes inactive and many devices terminate the current transaction.

60. Why must CS remain LOW during a transaction?
Because many SPI devices interpret CS HIGH as the end of the transaction and reset their internal communication state.

61. Which SPI signals are shared among multiple slaves?
MOSI
MISO
SCLK

62. Which signal must be unique for each slave?
CS (Chip Select)

 63. Why do unselected slaves not interfere with MISO?
Because they place MISO in a High Impedance (Hi-Z) state.

64. What happens if two CS lines are LOW simultaneously?
Multiple slaves drive MISO at the same time, causing bus contention and communication errors.

65. How does SPI select a slave?
By pulling that slave's CS pin LOW.

66. What is MISO contention?
 MISO contention happens when two selected slaves try to drive different bits onto the same MISO line at the same time, and SPI usually detects it only indirectly through corrupted data or signal analysis.

67. How does SPI indicate a read operation?
Using a READ command defined by the device datasheet.

68. Why is a dummy byte transmitted during SPI read?
To generate clock pulses so that the slave can transmit data.

69. Which line carries the received data?
MISO

70. Can the slave generate clock pulses?
No, Only the master generates SCLK.

71. What marks the start and end of an SPI read transaction?
Start → CS LOW
End   → CS HIGH

72. What is the purpose of SPI Write?
To send data from the master to the slave.

73. What is the typical SPI Write frame?
CS LOW
WRITE Command
Address/Register
Data
CS HIGH

74. Is a dummy byte required in SPI Write?
No, because actual data already generates clock pulses.

75. Which line carries write data?
MOSI

76. What marks the end of a write transaction?
CS = HIGH

77. What is a dummy byte?
A transmitted byte used only to generate clock pulses.

78. Why is a dummy byte required in SPI read?
Because the master must generate clock pulses for the slave to transmit data.

79. Which values are commonly used as dummy bytes?
0x00 or 0xFF

80. Is the dummy byte usually stored in the slave?
No, It is normally ignored.

81. Does I²C require dummy bytes?
No, because clock generation is independent of data transmission.

82. What does CS LOW indicate?
Slave selected and transaction active.

83. What does each clock pulse transfer?
1 MOSI bit
1 MISO bit

84. In Mode 0, on which edge is data sampled?
Rising Edge

85. Which four signals are normally observed on a logic analyzer?
CS
SCLK
MOSI
MISO

86. What are the first things to check when SPI communication fails?
CS
Clock
SPI Mode
MOSI
